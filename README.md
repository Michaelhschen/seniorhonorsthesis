# Thesis Data Processing Pipeline

This folder contains the empirical pipeline for the senior honours thesis,
*Real-Time Macroeconomic Regime Detection: The Role of GDP and Inflation
Nowcasting in Closing the Information Gap*. The code builds three comparable
macroeconomic information sets, applies a shared cleaning and PCA workflow,
fits Hidden Markov Models (HMMs), and compares the resulting regime
probabilities.

The pipeline is organized as four explicit stages so that each step can be
audited independently:

1. Collect and align macroeconomic data.
2. Run standalone cleaning and PCA diagnostics.
3. Build uniform PCA inputs and estimate HMMs.
4. Compare HMM outputs across information sets.

An optional fifth stage runs a small out-of-sample current-regime
classification check on the final holdout year.

Generated outputs are written to `outputs/`. They can be regenerated by running
the pipeline and do not need to be committed unless a frozen replication bundle
is desired.

## Repository Contents

Core scripts:

- `dataCollection.py`: builds the full-information, real-time, and
  nowcast-enhanced data panels.
- `dataCleaning.py`: runs standalone cleaning, stationarity tests,
  normalization, and PCA diagnostics.
- `hmmModeling.py`: rebuilds uniform PCA inputs and estimates the HMMs.
- `hmmModelComparison.py`: compares model outputs using internal and external
  benchmarks.
- `hmmOutOfSampleValidation.py`: optional holdout-year feasibility test using
  training-only scaling, PCA, and HMM estimation.

Manual input workbooks:

- `ISM Manufacturing PMI.xlsx`
- `Philly Fed BOS.xlsx`
- `ISM Manufacturing New Orders.xlsx` is included only if reproducing the
  candidate-variable testing history; it is not retained in the final model.

Package files:

- `requirements.txt`
- `requirements-analysis.txt`

## Environment Setup

Run commands from the project root, the folder containing `Data Processor`.
Paths with spaces should remain inside quotation marks.

Windows PowerShell:

```powershell
cd "<PROJECT_ROOT>"
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r "Data Processor\requirements.txt"
$env:FRED_API_KEY="<YOUR_FRED_API_KEY>"
```

macOS/Linux:

```bash
cd "<PROJECT_ROOT>"
python3 -m venv .venv
./.venv/bin/python -m pip install -r "Data Processor/requirements.txt"
export FRED_API_KEY="<YOUR_FRED_API_KEY>"
```

The FRED API key should be supplied through the environment. Do not hard-code
API keys into the scripts.

## Pipeline Run

The final thesis run uses a deterministic publication-lag real-time panel and
skips live NBER benchmark downloads.

Windows PowerShell:

```powershell
.\.venv\Scripts\python.exe "Data Processor\dataCollection.py" --real-time-mode lag
.\.venv\Scripts\python.exe "Data Processor\dataCleaning.py"
.\.venv\Scripts\python.exe "Data Processor\hmmModeling.py" `
  --refresh-pca-inputs `
  --pca-components 3,5 `
  --pca-output-components 5 `
  --state-counts 4 `
  --bic-diagnostic-state-counts 3,4,5
.\.venv\Scripts\python.exe "Data Processor\hmmModelComparison.py" `
  --pca-components 3,5 `
  --skip-nber-download
```

Optional out-of-sample current-regime validation:

```powershell
.\.venv\Scripts\python.exe "Data Processor\hmmOutOfSampleValidation.py" `
  --holdout-months 12 `
  --pca-components 3,5
```

macOS/Linux:

```bash
./.venv/bin/python "Data Processor/dataCollection.py" --real-time-mode lag
./.venv/bin/python "Data Processor/dataCleaning.py"
./.venv/bin/python "Data Processor/hmmModeling.py" \
  --refresh-pca-inputs \
  --pca-components 3,5 \
  --pca-output-components 5 \
  --state-counts 4 \
  --bic-diagnostic-state-counts 3,4,5
./.venv/bin/python "Data Processor/hmmModelComparison.py" \
  --pca-components 3,5 \
  --skip-nber-download
```

Optional out-of-sample current-regime validation:

```bash
./.venv/bin/python "Data Processor/hmmOutOfSampleValidation.py" \
  --holdout-months 12 \
  --pca-components 3,5
```

Use `--real-time-mode lag` for the thesis replication run. The alternative
vintage mode requires live ALFRED/FRED vintage access and may depend on the
availability of current API responses.

The optional out-of-sample script trains scaler, PCA, and HMM parameters only
through the pre-holdout sample, then recursively filters the final holdout
months. It compares those holdout probabilities with the existing full-sample
full-information HMM benchmark. This is a feasibility check for current-regime
classification, not a one-month-ahead recession forecast or trading backtest.

## Data Sources

The pipeline combines public macroeconomic and nowcasting sources:

- FRED/ALFRED macroeconomic series.
- Atlanta Fed GDPNow tracking workbook.
- Cleveland Fed inflation-nowcasting chart-data files.
- ISM Manufacturing PMI workbook.
- Philadelphia Fed Business Outlook Survey workbook.
- NBER recession chronology for the external benchmark.

Nowcast replacement is deliberately narrow. The nowcast-enhanced information
set replaces stale or unavailable real-time values for:

- `GDP_QoQAnn`
- `CPI_YoY`
- `CorePCE_YoY`

The nowcast model does not receive additional predictor columns. It uses the
same final variables, sample, PCA dimension, HMM state count, and covariance
restriction as the other information sets.

## Information Sets

The empirical design compares three information environments:

- **Full information:** latest available macroeconomic data, used as the ex
  post internal benchmark.
- **Real time:** values approximating what would have been available at each
  historical month-end date.
- **Nowcast enhanced:** the real-time panel with GDP and inflation nowcasts
  substituted where current official observations were unavailable.

The raw aligned panels contain 141 monthly rows from January 2014 through
September 2025. After uniform transformations, stationarity treatment, and
sample alignment, the HMM-ready sample contains 137 monthly observations from
May 2014 through September 2025.

## Final Model Design

The final HMM-ready panel uses 13 transformed macro-financial variables:

- `GDP_QoQAnn`
- `CorpCapex_QoQ_diff`
- `CorpFixedCap_QoQ_diff`
- `DelinqChange_diff`
- `CPI_YoY_diff`
- `CorePCE_YoY_diff`
- `UNRATE`
- `DXY_YoY_diff`
- `TermSpread_10Y3M_diff`
- `VIX_log`
- `M1M2_Ratio_diff`
- `ISM_PMI`
- `PhillyFed_BOS_diff`

The main model uses:

- static PCA factors for HMM estimation;
- rolling PCA only as a loading-stability diagnostic;
- five PCA factors in the main HMM;
- three PCA factors as an interpretability and sensitivity check;
- four hidden regimes;
- diagonal covariance;
- BIC diagnostics over 3-, 4-, and 5-state alternatives.

The four HMM regimes are labeled after estimation using PCA means, transition
behavior, and economic profiles:

- Expansion / Goldilocks
- Overheating / Late Cycle
- Contraction / Recession
- Recovery / Early Cycle

HMM state numbers are arbitrary, so analysis should use the economic labels
rather than raw state IDs.

## Interpretation

The internal benchmark compares real-time and nowcast-enhanced HMM probability
vectors against the full-information HMM probability vector. In the thesis run,
the 5-PC nowcast-enhanced model reduces the internal information gap by roughly
82.9 percent relative to the real-time baseline.

The NBER comparison is treated as a secondary external check, not the central
proof. The final sample contains only two NBER recession months, so those
metrics should be interpreted cautiously.

## Reproducibility Notes

- `outputs/` and `raw_downloads/` are generated or cached artifacts.
- Commit final outputs only if the repository is intended to serve as a frozen
  thesis replication package.
- For a lighter code repository, commit the scripts, README, requirements, and
  required manual workbooks; regenerate outputs by running the pipeline.
- If live downloads fail, use cached `raw_downloads/` files from a frozen run or
  rerun once internet/API access is available.

## Troubleshooting

If FRED/ALFRED downloads fail, confirm that `FRED_API_KEY` is set in the active
terminal session and that internet access is available.

If HMM comparison inputs are missing, rerun `hmmModeling.py` with
`--refresh-pca-inputs` and inspect `comparison_missing_inputs.csv`.

If paper or presentation numbers do not match the CSV outputs, rerun all four
pipeline stages and regenerate the figures from the current comparison outputs.
